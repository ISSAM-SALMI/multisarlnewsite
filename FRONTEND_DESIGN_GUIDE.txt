▓▓▓ ANIMATIONS PERSONNALISÉES ▓▓▓

fade-in-up
  • Durée: 1s
  • Easing: ease-out
  • De: opacity 0, translateY(20px)
  • Vers: opacity 1, translateY(0)
  • Utilisation: Éléments du Hero avec délais échelonnés (0.1s, 0.2s, 0.4s, etc.)

pulse-slow
  • Durée: 4s
  • Easing: cubic-bezier(0.4, 0, 0.6, 1)
  • Répétition: Infinie
  • Utilisation: Badge de statut "Intervention 24/7"

animate-ping
  • Animation Tailwind native
  • Utilisation: Point rouge clignotant dans le badge de statut

▓▓▓ TRANSITIONS COURANTES ▓▓▓

Durées Standard
  • Ultra-rapide: 150ms
  • Rapide: 300ms
  • Normale: 500ms
  • Lente: 700ms à 1000ms (images, overlays)

Types de Transitions
  • colors: Changements de couleur (boutons, liens)
  • all: Tout (utilisé pour effets complexes)
  • transform: Transformations (scale, translate)
  • opacity: Changements d'opacité

Effets Hover Récurrents
  • Boutons: Translation vers le haut + ombre accentuée
  • Icônes: Scale 110% (agrandissement 10%)
  • Images: Scale 110% + overlay sombre
  • Liens: Couleur change + soulignement animé
  • Cartes: Ombre étendue + bordure colorée

▓▓▓ BACKDROP BLUR ▓▓▓

Utilisation: Navbar scrollé, badges, cartes semi-transparentes
  • Classe: backdrop-blur ou backdrop-blur-md
  • Effet: Flou gaussien de l'arrière-plan pour effet de verre dépoli moderne

▓▓▓ GRADIENTS ▓▓▓

Dégradé Hero
  • Type: bg-gradient-to-r (de gauche à droite)
  • De: navy-950 (opaque) → navy-900/95 → navy-900/20 (transparent)
  • Effet: Masque l'image pour créer un espace de contenu lisible

Dégradé Texte
  • Type: bg-gradient-to-r avec bg-clip-text
  • De: white → gray-500
  • Effet: Texte avec couleur dégradée (ligne 2 du titre Hero)

Brillance Bouton
  • Type: Pseudo-élément avec gradient horizontal
  • Mouvement: -translate-x-full → translate-x-full
  • Durée: 1000ms
  • Effet: Bande lumineuse qui traverse le bouton au hover

▓▓▓ OMBRES (SHADOWS) ▓▓▓

Hiérarchie d'Ombres
  • shadow-sm: Ombre très légère (cartes au repos)
  • shadow-md: Ombre moyenne (tooltips, dropdowns)
  • shadow-lg: Ombre large (navbar, éléments importants)
  • shadow-xl: Ombre extra-large (cartes au hover)
  • shadow-2xl: Ombre maximale (formulaire de contact)

Ombres Colorées
  • shadow-black/20: Ombre noire à 20% d'opacité
  • shadow-red-900/40: Ombre rouge sombre (bouton CTA)
  • shadow-red-700/50: Ombre rouge au hover
  • shadow-red-200: Ombre rouge claire (bouton formulaire)

▓▓▓ BREAKPOINTS TAILWIND ▓▓▓

  • sm: 640px (Petits smartphones en paysage)

  • xl: 1280px (Grands écrans)
  • 2xl: 1536px (Très grands écrans)

▓▓▓ STRATÉGIES RESPONSIVES ▓▓▓

Mobile First
  • Classes sans préfixe = Mobile par défaut
  • md:, lg:, xl: appliquent des styles pour écrans plus larges

Typographie
  • Titres H1: text-4xl (mobile) → md:text-6xl → lg:text-7xl
  • Titres H2: text-3xl → md:text-4xl
  • Corps: text-base → md:text-lg
  • Espacement: mb-4 → md:mb-6 → lg:mb-8

Layout
  • Grilles: grid-cols-1 (mobile) → md:grid-cols-3 ou md:grid-cols-4
  • Flex: flex-col (empilé mobile) → md:flex-row (côte à côte)
  • Gaps: gap-4 → md:gap-6 → lg:gap-8

Espacements
  • Padding: px-4 (mobile) → md:px-6 (desktop)
  • Sections: py-16 → md:py-24

Visibilité
  • Éléments cachés mobile: hidden md:block (barre de stats Hero)
  • Menu mobile: visible mobile, caché desktop (lg:hidden)
  • Navigation desktop: hidden lg:flex

Images & Médias
  • Hauteur Hero: min-h-[700px] pour garantir une hauteur minimale
  • Position image Hero: Centrée mobile, décalée droite desktop

▓▓▓ NAVIGATION CLAVIER ▓▓▓

  • Focus visible: Tous les éléments interactifs ont focus:ring
  • Ordre de tabulation: Logique et séquentiel
  • Bouton menu mobile: aria-label="Toggle Menu"

▓▓▓ LABELS SÉMANTIQUES ▓▓▓

  • Logo SVG: aria-label="Logo Multiresaux"
  • Bouton WhatsApp: aria-label="Contact sur WhatsApp"
  • Liens: Texte descriptif ou aria-label

▓▓▓ CONTRASTES ▓▓▓

  • Texte sur fond clair: navy-900 (#0A192F) → Ratio > 7:1 (AAA)
  • Texte sur fond sombre: white (#FFFFFF) → Ratio maximal
  • Liens: Changement de couleur au hover pour feedback visuel

▓▓▓ ÉTATS INTERACTIFS ▓▓▓

  • Hover: Changement visible (couleur, échelle, ombre)
  • Active: Feedback immédiat
  • Focus: Bordure ou ring de focus visible
  • Disabled: Opacité réduite + cursor-not-allowed

▓▓▓ SCROLL BEHAVIOR ▓▓▓

  • Navigation anchor: Scroll fluide (scroll-smooth)
  • Lock scroll: Body overflow hidden quand menu mobile ouvert
  • Sticky navbar: Fixed position pour toujours accessible

Tailwind CSS
  • CDN: cdn.tailwindcss.com
  • Configuration inline: Pas de build séparé
  • Production: Devrait être compilé pour purge des classes inutilisées

Images
  • Source: Unsplash (CDN externe)
  • Format: Paramètres d'optimisation (w=, q=, auto=format)
  • Lazy loading: Recommandé (non implémenté actuellement)

▓▓▓ ANIMATIONS ▓▓▓

GPU Acceleration
  • Transform: Utilise GPU (translate, scale, rotate)
  • Opacity: Optimisé par navigateur
  • Évite: left, top, width, height pour animations

Will-change
  • Non utilisé actuellement
  • Pourrait être ajouté sur éléments animés fréquemment

React 19
  • Strict Mode: Devrait être activé en dev
  • Memo: Pourrait optimiser composants purs
  • Lazy loading: Composants code-split si nécessaire

                         13. CONVENTIONS & BONNES PRATIQUES

▓▓▓ NOMMAGE CSS (TAILWIND) ▓▓▓

Structure des Classes
  1. Layout (display, position, flex, grid)
  2. Espacement (margin, padding)
  3. Taille (width, height)
  4. Typographie (font, text)
  5. Couleurs (bg, text, border)
  6. Effets (shadow, rounded, opacity)
  7. Transitions & animations
  8. États (hover:, focus:, group-hover:)

Ordre Responsive
  • Base (mobile) → md: → lg: → xl:

▓▓▓ COMPOSANTS REACT ▓▓▓

Structure
  • Imports en haut
  • Interfaces/Types
  • Composant avec typage TypeScript
  • Export en bas ou inline

Nommage
  • PascalCase pour composants
  • camelCase pour fonctions/variables
  • SCREAMING_SNAKE_CASE pour constantes

Props
  • Interface explicite pour chaque composant
  • Props optionnelles marquées avec ?

▓▓▓ ORGANISATION FICHIERS ▓▓▓

Structure Actuelle
  /components (tous les composants UI)
  /types.ts (types partagés)
  App.tsx (composant racine)
  index.tsx (entry point)

Recommandations
  • Garder composants unitaires
  • Partager types communs
  • Éviter duplication de code
